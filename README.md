# gents

A tiny Go→TypeScript types generator. Reads Go source files, finds structs
marked with `//gents:export`, and emits matching TypeScript interfaces plus
zero-value factory functions.

The guiding rule: **gents emits the TS shape that matches what
`encoding/json.Marshal` produces on the wire**. No other struct tags read.
No configuration files. No third-party dependencies.

Full reference: [docs/gents_package.md](docs/gents_package.md).

## Why gents

- **Predictable factory zeros.** `newItem()` produces a value with
  empty-string scalars, `null` pointers, `[]` slices, `{}` maps —
  matching what JS code already treats as "not set yet." `[]byte` zeros
  to `''` (the empty base64 string), `int64` tagged `,string` zeros to
  `'0'`. Forms scaffolded from the factory feel like blank inputs, not
  AD-1 placeholders.
- **`encoding/json` fidelity** where competing tools get edge cases
  subtly wrong. `[]byte → string (base64)` and `,string` coercion are the
  two most common footguns and both are handled correctly.
- **Pure stdlib, no dependencies**. Install it, use it, forget it.
  Nothing to audit.

## Install

```
go install github.com/mmalcek/gents/cmd/gents@latest
```

## Quickstart

```go
// item.go
package example

import "time"

//gents:export
type Item struct {
    ID          string    `json:"id"`
    Name        string    `json:"name"`
    Description string    `json:"description,omitempty"`
    Tags        []string  `json:"tags"`
    CreatedAt   time.Time `json:"created_at"`
}
```

```
gents -in item.go -out item.ts
```

```typescript
// Code generated by github.com/mmalcek/gents; DO NOT EDIT.

export interface Item {
  id: string
  name: string
  description?: string
  tags: string[]
  created_at: string
}

export function newItem(): Item {
  return {
    id: '',
    name: '',
    tags: [],
    created_at: '',
  }
}
```

Wire it into `go generate`:

```go
//go:generate go run github.com/mmalcek/gents/cmd/gents@v0.1.0 -in item.go -out ../client/types/item.ts
```

Pin to a tag — `@latest` may churn generated output on future releases.

## Programmatic use

```go
import "github.com/mmalcek/gents"

src, err := gents.Generate("item.go", gents.Options{})
if err != nil {
    return err
}
_ = os.WriteFile("item.ts", []byte(src), 0o644)
```

Bundle mode takes a directory instead:

```go
src, err := gents.GenerateDir("api/", gents.Options{
    TypeMap: map[string]string{"uuid.UUID": "string"},
})
```

See [pkg.go.dev](https://pkg.go.dev/github.com/mmalcek/gents) for the full
library reference and runnable examples.

## CLI reference

```
gents [-in <path>] -out <output.ts> [flags]

  -in string      Go source file or directory to process. Directories
                  are walked recursively. Defaults to the current
                  directory (".") when omitted.
  -out string     TypeScript file to write (required)
  -strip string   Prefix stripped from Go struct names when building the
                  factory function name (default "t" — turns marked
                  struct tFoo into factory newFoo). The TS interface
                  name itself is always emitted verbatim. Pass
                  -strip="" to keep the prefix on the factory too.
  -force          Overwrite -out even if it wasn't generated by gents
                  (default: refuse, to protect hand-written .ts files)
  -map KEY=VAL    Custom Go-to-TS type mapping (e.g. uuid.UUID=string).
                  Repeatable. Factory zero inferred from TS type.
```

Three invocation patterns:

```
gents -out types.ts                    # bundle the current directory
gents -in api/ -out api.ts             # bundle a subtree
gents -in item.go -out item.ts         # single file
```

gents stat's `-in` to decide: file path → single-file mode, directory →
bundle mode. The overwrite guardrail refuses to clobber a file whose first
line isn't the gents-generated header; pass `-force` if you really mean it.
Parent directories of `-out` are created automatically.

## Bundle mode (cross-file references)

When the path you point at is a directory, gents walks it recursively
(skipping `_test.go`, `testdata/`, `_`-prefixed, and `.`-prefixed entries —
Go-toolchain conventions), parses every `.go` file, and emits one TS file
covering every marked struct in sorted-path order. Cross-file references
resolve because everything lands in the same output. Two marked structs
that strip to the same TS name panic with both source locations.

## Named aliases auto-resolve

Named primitive aliases declared in the scanned input resolve on their
own — no `-map` needed:

```go
type UserID string
type Score int

//gents:export
type User struct {
    ID    UserID `json:"id"`     // emits as string
    Score Score  `json:"score"`  // emits as number
}
```

Caveat: if your type declares a `MarshalJSON` method, the wire shape
isn't the underlying type — gents panics with a hint instead of
guessing wrong. Supply `-map` to override.

Cross-package types (`uuid.UUID` and friends) still need `-map`: gents
doesn't load external packages, and even if it did, resolving based on
underlying types would be wrong for types with custom `MarshalJSON`
(almost all of them).

## Custom type mappings

For third-party types, qualified selectors, or any Go type gents can't
figure out, declare the TS shape — either via CLI flag or via a
co-located source directive:

```
gents -in api.go -out api.ts \
  -map uuid.UUID=string \
  -map decimal.Decimal=string \
  -map "time.Time=Date | null"
```

or, equivalently, in the source file itself:

```go
//go:generate go run github.com/mmalcek/gents/cmd/gents@v0.1.0 -out ../client/types.ts

//gents:map uuid.UUID=string
//gents:map decimal.Decimal=string
//gents:map time.Time=Date | null

package api
```

Source directives are **global across the bundle** — declared in one
file, applied to references in every file. No duplication. CLI `-map`
still works and overrides directives silently (explicit runtime choice
wins). Two directives disagreeing about the same Go type panic with
both locations.

User mappings override built-ins — if you'd rather have `time.Time` emit
as `Date | null` instead of an RFC3339 string, the `-map` above does it.
Pointer, slice, and map wrapping apply naturally by recursion: with
`-map uuid.UUID=string` in place, `*uuid.UUID` → `string | null`,
`[]uuid.UUID` → `string[]`, `map[string]uuid.UUID` → `Record<string,
string>`.

Factory zeros are inferred from the TS type (`string` → `''`, `X | null`
→ `null`, `X[]` → `[]`, etc.); TS types whose zero can't be inferred
(e.g. plain `Date`) panic at emit time with a hint to add `| null`.

## JSON tag modifiers

gents understands the same set `encoding/json` does:

| Tag | Effect on TS output |
|---|---|
| `json:"name"` | Field emitted with the given wire name. |
| `json:"-"` | Field omitted entirely. |
| `json:"name,omitempty"` | Field marked optional (`?:`); factory omits it. |
| `json:"name,omitzero"` (Go 1.24+) | Same as `omitempty` for TS purposes. |
| `json:"n,string"` | Coerces numeric/boolean fields to TS `string` (matches JSON-string wire format, used for JS-safe `int64` IDs). |

Unknown flags panic. `,string` on anything other than a numeric or boolean
base type panics — `encoding/json` itself ignores it there.

## Type mapping

| Go | TypeScript | Factory zero |
|---|---|---|
| `string` | `string` | `''` |
| `bool` | `boolean` | `false` |
| `int*`, `uint*`, `float*`, `byte`, `rune` | `number` | `0` |
| `time.Time` | `string` (RFC3339 on the wire) | `''` |
| `time.Duration` | `number` (nanoseconds) | `0` |
| `json.RawMessage` | `unknown` | `null` |
| `any` / `interface{}` | `unknown` | `null` |
| `[]byte` / `[]uint8` | `string` (base64) | `''` |
| `[]T` (other) | `T[]` | `[]` |
| `map[string]V` | `Record<string, V>` | `{}` |
| `*T` | `T \| null` | `null` |
| Sibling `//gents:export` struct | verbatim interface name | `new<StrippedName>()` |
| Any type declared via `-map` / `Options.TypeMap` | as specified | inferred |

JSON names that aren't valid TS identifiers (`content-type`, `123status`,
etc.) are automatically quoted in both the interface and the factory.

## Limitations

- Bundle mode emits one TS output file; per-file mirror output is not
  supported (bundle already covers the functional case; modern TS
  bundlers tree-shake at the symbol level regardless of file layout).
- Only `json:` tags are read. `gorm`, `binding`, `validate`, `xml`,
  `msgpack` — all invisible to gents.
- Embedded (anonymous) fields: `Base `json:"-"`` skips them, `Base
  `json:"name"`` emits them as nested objects, and an untagged
  embed flattens Base's fields into the outer struct (matches
  `encoding/json`, including the least-nested / tagged-wins
  dominant-field rules; `*Base` makes contributed fields optional).
  Cross-package embedding (`gorm.Model`) still panics — declare a
  local mirror or use `-map` for a TS placeholder.
- Unsupported types panic with a `file:line` pointer: fixed-length
  arrays (`[N]T`), double pointers (`**T`), inline anonymous structs,
  interfaces with non-empty method sets, `chan`, `func`. Third-party
  or named-alias types resolve via `-map`.
- No Request/Response split from a single struct. Mark one struct, get
  one interface; mark two, get two. The tool takes no position on that
  architectural choice — see [docs/gents_package.md §1](docs/gents_package.md)
  for the reasoning.
